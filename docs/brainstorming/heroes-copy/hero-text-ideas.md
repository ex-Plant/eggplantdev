# Hero Text Ideas

Agent: Claude

## Overall Direction

- **Tone:** Cosmic / mystical — but with Fallout-style retro-futuristic specimen catalog energy
- **Register:** Each eggplant hero is like a page from a classified cosmic produce catalog — labeled, categorized, presented with deadpan institutional seriousness
- **Structure:** subtitle (small monospace) + two-line title + description + optional button(s)
- **Key word:** Singularity — works on two levels: cosmic physics (point from which everything emerges) and portfolio identity (one-of-a-kind)
- **Story logic:** Each hero plays a distinct narrative role (per portfolio memory doc), not all equally mystical
- **Naming system:** "Object #NNN" prefix in subtitles (preferred over "Specimen" — scales better, stronger, better programming double-meaning)
- **Title pattern:** "The / \_\_\_" as default, with deliberate breaks at emotional turning points (#003 and #005)
- **Page order:** encounter → structure → obsession → work → work → person → principles → exit

## Key Words

Dual-meaning words — cosmic/sci-fi AND programming. This is the linguistic bridge between the surreal eggplant universe and the developer portfolio.

| Word        | Cosmic Meaning                          | Programming Meaning                                     | Best Fit Hero                                          |
| ----------- | --------------------------------------- | ------------------------------------------------------- | ------------------------------------------------------ |
| Singularity | Point of infinite density, cosmic event | One-of-a-kind, unique system                            | Opening hero — the grand entrance                      |

| Specimen    | Classified artifact, catalog entry      | Test subject, the thing under observation               | Alternative to "Object" (more charming, less scalable) |

## The Full Catalog

### Page Sequence

```
1.  Object #000 — Space / Oddity          → encounter (what is this?)
2.  intro text
3.  Object #001 — The / Blueprint         → structure (it's built on systems)
4.  Team Work section
5.  Object #002 — Peculiar / Shrine       → obsession (the craft became ritual)
6.  Freelance section
7.  Object #003 — Solar / Entity          → the person (who made this?)
8.  About section
9.  Object #004 — The / Doctrine          → principles (what do they believe?)
10. Values section
11. Object #005 — Return / Void           → exit (the signal stays open)
12. Contact / Footer
```

**Why this order:**

- Work (Team + Freelance) comes first — proves capability
- About before Values — identity gives context, values without a person are slogans
- Values after About — principles are more believable once you know who's stating them
- Contact last — natural hiring funnel: work → person → trust → reach out

encounter → structure → obsession → [team work] → [freelance] → person → principles → exit

The first three heroes are cold catalog entries. #003 warms up (the system meets the person). #004 returns to authority (the manifesto). #005 releases (the function returns void).

## Object #000 — Space Oddity (Opening Hero)

**Role:** First thing visitors see. Stop the scroll. Establish the universe. One button that scrolls to next section.
**Visual:** glamCosmicBillboard (Soleil Aubergine slot used for testing)

```
subtitle: "Object #000"
titleLine1: "Space"
titleLine2: "Oddity"
description: "Less vegetable, more minor celestial authority. Warm, theatrical,
              impossible to ignore, and somehow still tasteful."
button: TBD
```

**Status:** Title and description locked. Button TBD.

**Why "Space Oddity":** Bowie reference. The eggplant IS the oddity — the strange object that doesn't belong but commands attention. Works on multiple levels: cultural, cosmic, portfolio identity.

**Why this description works as the opener:** Sets the tone immediately — absurd, confident, self-aware. "Minor celestial authority" is the perfect introduction to the whole catalog.

### Alternative Descriptions (explored, not selected)

**Deadpan Specimen Report:**

> First observed during a routine frontend deployment. Origin unclear. Properties include: geometric precision, suspicious luminosity, and an unusual affinity for clean architecture. Further study ongoing.

**Cosmic Prestige:**

> A prestige artifact of uncertain origin and undeniable presence. Distilled from sacred geometry, starlight, and the kind of engineering confidence that ships the strange idea beautifully.

**Transmission / Signal:**

> The signal arrived without warning — one glossy object at the center of an impossible geometry. It brought with it a portfolio of improbable builds, clean code, and suspiciously good taste.

**Catalog Entry (Most Fallout-like):**

> Classification: celestial produce, engineering-grade. First documented in the wild spaces between design and deployment. Known properties: sacred geometry alignment, full-stack capability, mild gravitational pull. Handle with reverence.

### Button Ideas

Single button, scrolls to next section.

- `Enter the Orbit`
- `Begin Observation`
- `Open Catalog`
- `Proceed`
- `View Transmission`
- `Approach the Object`

## Object #002 — Peculiar Shrine (Hex Lattice)

**Role:** Craft / precision / personal method. Pairs with Freelance section. Marks the shift from shared systems to individual control.

```
subtitle: "Object #002"
titleLine1: "Peculiar"
titleLine2: "Shrine"
description: "At some point the layout stopped being a layout and became a shrine.
              Since then the eggplant has remained at the center, supervising
              alignment and other minor miracles."
```

**Status:** Title and description locked.

**Why "Peculiar" over "Celestial":** "Celestial Shrine" was the old default variant title — no evolution. "Peculiar" breaks the cosmic register just enough to feel authored. It admits the whole thing is a bit weird without apologizing. Museum/archive quality fits the catalog system.

**Why this description won:** "The layout stopped being a layout" is immediately legible — everyone knows what a layout is. The transformation to "shrine" is clear. "Supervising alignment and other minor miracles" is the strongest closing line in the set.

## Object #003 — Solar Entity (Soleil Aubergine)

**Role:** Persona / warmth / the person behind the work. Pairs with About section. This is where the catalog **warms up**. The first three objects are cold/structural — this one is the anomaly the system can't classify.

```
subtitle: "Object #003 / Classification: pending"
titleLine1: "Solar"
titleLine2: "Entity"
description: TBD — choosing between options
```

**Why "Solar / Entity" breaks the "The" pattern:** The catalog glitches when it encounters something warm. The naming system falters. That's the point — the person behind the portfolio doesn't fit neatly into the index.

### Description Options

**Option A — System failure report:**

> Not all objects in the catalog conform to specification. This one radiates warmth, maintains eye contact across frameworks, and has never once returned null.

**Option B — Classification attempt:**

> The system attempted classification and returned: inconclusive. Properties include: persistence across environments, unreasonable warmth, and an unexplained affinity for both sacred geometry and functional components.

**Option C — Personal reveal (most human, ties to DJ identity):**

> Entity entered the catalog without authorization. Origin: terrestrial. Known behaviors: building things that work, caring more than strictly necessary, playing djembe at inappropriate hours.

**Recommendation:** Option C. Drops the mask slightly, lets the real person through. "Playing djembe at inappropriate hours" ties to the Eggplant Man / Djembeya identity from the design spec.

## Object #004 — The Doctrine (Cosmic Cult Flyer)

**Role:** Values / manifesto / belief system. Pairs with Values section. The catalog stops documenting and starts recruiting.

```
subtitle: "Object #004 / Status: non-negotiable"
titleLine1: "The"
titleLine2: "Doctrine"
description: TBD — choosing between options
```

### Description Options

**Option A — Minimal recruitment:**

> No robes. No initiation ritual. Just an unreasonable commitment to clean interfaces, honest code, and the persistent suspicion that the eggplant is watching.

**Option B — Consequences included:**

> These are the operating principles. They were not voted on. They do not expire. Violators will be reassigned to a codebase without types.

**Option C — The full doctrine (recommended):**

> The order asks very little. Ship with intention. Respect the grid. Never deploy on Friday. Acknowledge the eggplant. That is the entire doctrine.

**Recommendation:** Option C. "Never deploy on Friday" is a universal dev joke that immediately signals "this person actually builds things" — grounding right where the manifesto energy needs it most.

## Object #005 — Return Void (Cosmic Flower / Epilogue)

**Role:** Closing / contact / the signal stays open. Pairs with contact/footer. The catalog closes.

```
subtitle: "Object #005 / Signal: open"
titleLine1: "Return"
titleLine2: "Void"
description: TBD — choosing between options
```

**Why "Return / Void":** `return void` = the function ends, the portfolio ends, the cosmic journey ends. But "return" also means: come back. Call to action embedded in a programming pun. Breaks the "The" pattern again — the catalog is no longer labeling, it's releasing you.

### Description Options

**Option A — Catalog closure:**

> The catalog is complete. The transmission remains active. If any of this resonated — or even if it didn't — the signal is open.

**Option B — All systems nominal:**

> All objects accounted for. All orbits stable. The eggplant continues its trajectory. Coordinates below.

**Option C — Terminal output (recommended):**

> End of index. The function returns void, but the channel stays open. Frequency: available. Response time: human.

**Recommendation:** Option C. "The function returns void, but the channel stays open" is the entire closing statement. "Response time: human" is the perfect last line.
